I also noticed a crucial detail in the project specifications: the professor is specifically awarding a bonus point based on the **F1 score for target = 1** on their hidden verification dataset. In credit risk, `target = 1` usually represents the "risky" or "defaulting" minority class. Because this class is almost certainly imbalanced (far fewer risky loans than reliable ones), standard accuracy is a deceptive metric, and out-of-the-box models will naturally struggle to achieve a high F1 score for it.

To aggressively target that F1-Score, we have three primary architectural levers we can pull.

### **1. Algorithmic Class Weighting (The Quickest Win)**

By default, models treat all errors equally. In an imbalanced dataset, the model simply learns to predict the majority class (0) to minimize overall error, completely destroying your recall (and thus, F1) for class 1. We must force the algorithms to penalize mistakes on class 1 more heavily.

* **XGBoost:** We use the `scale_pos_weight` parameter. The industry standard is to set this to the ratio of negative instances to positive instances (e.g., `num_class_0 / num_class_1`).

### **2. Decision Threshold Tuning (The Precision Instrument)**

Our current `predict` function uses the default decision threshold of `0.5`. If the predicted probability is > 0.5, it outputs class 1. However, for imbalanced datasets, the optimal threshold for the F1 score is rarely 0.5.

Instead of accepting the default, we can compute the probabilities (`predict_proba`) and iterate through thresholds (e.g., 0.1 to 0.9) to empirically find the exact threshold that maximizes the F1 score on the training/validation set, and then apply *that* specific threshold to the test set.

### **3. Hyperparameter Optimization (The Heavy Lifter)**

Once we handle the imbalance, we need to tune the internal mechanics of the models. Rather than guessing parameters, we should implement a search strategy targeting the F1 metric directly.

* **GridSearchCV / RandomizedSearchCV:** Standard `scikit-learn` tools.

import os
import time
import json
import argparse
import joblib
import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LogisticRegression
from xgboost import XGBClassifier
from sklearn.metrics import accuracy_score, precision_recall_fscore_support, roc_auc_score, confusion_matrix

def train(
    classifier: str,
    reducedTrain_csv: str,
    target_column: str,
    model_path: str,
    metrics_json: str,
    seed: int = 42
):
    """
    Trains the specified binary classifier on the reduced dataset and saves the model.
    Incorporates class weighting to optimize F1-score for the minority class.
    """
    # --- 1. Data Loading ---
    start_input_time = time.time()
    df = pd.read_csv(reducedTrain_csv)
    dataset_input_time = time.time() - start_input_time

    y = df[target_column]
    X = df.drop(columns=[target_column])

    # --- 2. Calculate Class Imbalance ---
    # For XGBoost: scale_pos_weight = sum(negative instances) / sum(positive instances)
    num_class_0 = (y == 0).sum()
    num_class_1 = (y == 1).sum()
    
    # Handle potential edge cases (e.g., extremely skewed or buggy test datasets)
    scale_pos_w = float(num_class_0 / num_class_1) if num_class_1 > 0 else 1.0

    # --- 3. Classifier Selection & Weighting ---
    clf_name = classifier.lower().strip()
    
    if clf_name in ["random_forest", "randomforest"]:
        # 'balanced' automatically adjusts weights inversely proportional to class frequencies
        model = RandomForestClassifier(random_state=seed, class_weight='balanced')
        
    elif clf_name in ["xgb", "xgboost"]:
        # Explicitly pass the calculated ratio for the positive class
        model = XGBClassifier(random_state=seed, eval_metric='logloss', scale_pos_weight=scale_pos_w)
        
    elif clf_name in ["logistic_regression", "logisticregression"]:
        model = LogisticRegression(random_state=seed, max_iter=1000, class_weight='balanced')
        
    else:
        raise ValueError(f"Unsupported classifier: {classifier}. Choose from: random_forest, xgboost, logistic_regression.")

    # --- 4. Model Training ---
    start_train_time = time.time()
    model.fit(X, y)
    training_time = time.time() - start_train_time

    # --- 5. Save Artifacts ---
    os.makedirs(os.path.dirname(model_path) or '.', exist_ok=True)
    os.makedirs(os.path.dirname(metrics_json) or '.', exist_ok=True)

    joblib.dump(model, model_path)

    n_samples = len(df)
    target_1_percentage = (num_class_1 / n_samples) * 100

    stats = {
        "classifier": classifier,
        "seed": seed,
        "training_dataset": os.path.basename(reducedTrain_csv),
        "target_column": target_column,
        "model_path": os.path.basename(model_path),
        "n_samples": n_samples,
        "n_features": X.shape[1],
        "target_1_percentage": round(target_1_percentage, 2),
        "dataset_input_time": round(dataset_input_time, 2),
        "training_time": round(training_time, 2)
    }

    with open(metrics_json, 'w') as f:
        json.dump(stats, f, indent=4)
        
    print(f"Training completed for {classifier} (Class Imbalance Handled). Model saved to {model_path}.")

This is a fantastic next step. Tuning the decision threshold is one of the most mathematically elegant ways to improve the F1 score without adding any complex feature engineering or heavy computation.

By default, algorithms like Random Forest and XGBoost predict `target = 1` only if the probability score is $\ge 0.5$. ### **How We Will Implement This**

import os
import time
import json
import argparse
import joblib
import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LogisticRegression
from xgboost import XGBClassifier
from sklearn.metrics import accuracy_score, precision_recall_fscore_support, roc_auc_score, confusion_matrix

def train(
    classifier: str,
    reducedTrain_csv: str,
    target_column: str,
    model_path: str,
    metrics_json: str,
    seed: int = 42
):
    """
    Trains the specified binary classifier on the reduced dataset and saves the model.
    Incorporates class weighting and dynamic threshold tuning to optimize F1-score.
    """

    start_input_time = time.time()

    dataset_input_time = time.time() - start_input_time

    X = df.drop(columns=[target_column])

    num_class_0 = (y == 0).sum()

    elif clf_name in ["xgb", "xgboost"]:

By combining all three optimizations, we are:

1. **(Opt 1) Weighting the classes** to force the algorithm to care about the minority `target = 1`.
2. **(Opt 3) Searching for the best hyperparameters** specifically targeting the `f1` metric via Cross-Validation.
3. **(Opt 2) Fine-tuning the decision threshold** on that newly optimized model to squeeze out the absolute maximum F1 score.

### **How We Will Implement This**

We will wrap our base models in `scikit-learn`'s `RandomizedSearchCV`. Instead of testing every single combination (which takes forever), `RandomizedSearchCV` samples a fixed number of parameter combinations (e.g., 10) and evaluates them using 3-fold cross-validation.

Crucially, we will set `scoring='f1'`. This tells the search algorithm: *"I do not care about overall accuracy; give me the parameters that maximize the F1 score for the positive class."*

def train(
    classifier: str,
    reducedTrain_csv: str,
    target_column: str,
    model_path: str,
    metrics_json: str,
    seed: int = 42
):
    """
    Trains the specified binary classifier on the reduced dataset.
    Incorporates Class Weighting, Hyperparameter Optimization, and Threshold Tuning.
    """
    # --- 1. Data Loading ---
    start_input_time = time.time()
    df = pd.read_csv(reducedTrain_csv)
    dataset_input_time = time.time() - start_input_time

    y = df[target_column]
    X = df.drop(columns=[target_column])

    # --- 2. Calculate Class Imbalance ---
    num_class_0 = (y == 0).sum()
    num_class_1 = (y == 1).sum()
    scale_pos_w = float(num_class_0 / num_class_1) if num_class_1 > 0 else 1.0

    # --- 3. Classifier Setup & Hyperparameter Grids (Optimization #3) ---
    clf_name = classifier.lower().strip()
    
    if clf_name in ["random_forest", "randomforest"]:
        base_model = RandomForestClassifier(random_state=seed, class_weight='balanced')
        param_grid = {
            'n_estimators': [50, 100, 200],
            'max_depth': [None, 10, 20, 30],
            'min_samples_split': [2, 5, 10]
        }
    elif clf_name in ["xgb", "xgboost"]:
        base_model = XGBClassifier(random_state=seed, eval_metric='logloss', scale_pos_weight=scale_pos_w)
        param_grid = {
            'learning_rate': [0.01, 0.1, 0.2],
            'max_depth': [3, 5, 7],
            'n_estimators': [50, 100, 200],
            'subsample': [0.8, 1.0]
        }
    elif clf_name in ["logistic_regression", "logisticregression"]:
        base_model = LogisticRegression(random_state=seed, max_iter=2000, class_weight='balanced')
        param_grid = {
            'C': [0.01, 0.1, 1.0, 10.0],
            'solver': ['lbfgs', 'liblinear']
        }
    else:
        raise ValueError(f"Unsupported classifier: {classifier}. Choose from: random_forest, xgboost, logistic_regression.")

    # Execute Randomized Search targeting the F1 score
    start_train_time = time.time()
    print(f"Starting Hyperparameter Optimization for {classifier}...")
    
    search = RandomizedSearchCV(
        estimator=base_model,
        param_distributions=param_grid,
        n_iter=10,        # Number of parameter settings that are sampled
        scoring='f1',     # Explicitly optimize for F1 score
        cv=3,             # 3-fold cross-validation
        random_state=seed,
        n_jobs=-1         # Use all available CPU cores

            if f1[1] > best_f1:

                best_threshold = t

    # --- 5. Save Artifacts ---
    os.makedirs(os.path.dirname(model_path) or '.', exist_ok=True)
    os.makedirs(os.path.dirname(metrics_json) or '.', exist_ok=True)
